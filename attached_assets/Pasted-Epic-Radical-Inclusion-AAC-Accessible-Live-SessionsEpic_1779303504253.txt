Epic: Radical Inclusion – AAC-Accessible Live SessionsEpic Description:As an inclusive facilitator, I want KwaCart to natively support Alternative and Augmentative Communication (AAC) workflows, devices, and pacing, so that non-verbal, switch-scanning, and eye-gaze users can actively participate in live sessions without physical or temporal exclusion.User Story 1: Visual Pacing & Reduced Eye-Fatigue (Participant)As a participant using an eye-gaze tracker or switch-scanning system,I want to toggle a static, high-contrast user interface with an abstract time indicator,So that I can compose my thoughts without layout changes or ticking countdowns pulling my focus and causing physical fatigue.Acceptance Criteria:Given a live session page, when I toggle "AAC Mode," the visual countdown timer must replace numeric ticking clocks with a static, non-animated color indicator (Green $\rightarrow$ Amber $\rightarrow$ Red).The page layout must freeze entirely—no floating banners, pop-up "saved" notices, or dynamic resizing.All interactive targets (input box, submit button) must increase to a minimum target size of $80 \times 80$ pixels with a minimum of $24\text{px}$ padding between elements.User Story 2: The Composition Presence Flag (Facilitator Dashboard)As a session host,I want to see a visual indicator on my dashboard when an AAC user is actively drafting a response—even if their browser input field is currently empty,So that I do not prematurely close a session and cut off a voice while they are mid-composition.Acceptance Criteria:Given a participant using an external communication application (e.g., Grid 3), when they interact with their device keyboard while focused on the KwaCart tab, the app must emit an active metadata "heartbeat" to the server.The Facilitator Dashboard must display a dynamic status label next to that user's name: [User is Composing...].If the facilitator clicks "Close Session" while an active composition heartbeat is detected, a warning modal must appear: "An AAC participant is still composing. Do you still want to close?"User Story 3: Non-Text Submissions & Facilitator Synthesis (Workflow)As an AAC symbol-based user or voice-output device user,I want to submit my contribution as an audio clip or an image snapshot of my communication board,So that I can express my ideas using my primary vocabulary method rather than an alphabet keyboard.Acceptance Criteria:The participant input form must offer three distinct, large-target toggle tabs: [Text], [Audio (Record Device Speak)], and [Upload Image (Board Screenshot)].When a non-text file is submitted, it must bypass the automated text aggregation engine and route to a temporary "Facilitator Review Staging Area."Before generating the final Markdown (.md) summary, the facilitator or support worker must be provided an inline text field adjacent to the audio/image file to manually transcribe or translate the user's input.The final markdown file must render the manual transcription inline, seamlessly maintaining the text flow of the workshop output.
