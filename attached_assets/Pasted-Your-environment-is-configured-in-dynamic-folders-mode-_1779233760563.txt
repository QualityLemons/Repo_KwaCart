Your environment is configured in dynamic folders mode. In this mode, the asset's folder (asset_folder) is independent of the slashes in the public_id.

Your Django code sets public_id = "archives/md/..." and public_id = "archives/rtf/...". Those slashes form a path inside the public_id, not an asset folder. Unless your RawMediaCloudinaryStorage upload call explicitly sets an asset_folder parameter (e.g., asset_folder="archives/md"), the uploaded assets will land in the home folder with no asset folder set.

This affects which trigger filter to use:

    Option A — Filter by asset_folder (the recommended key in dynamic mode): Only works if your upload call explicitly sets asset_folder to archives/md or archives/rtf. If it doesn't, the flow will never trigger.
    Option B — Filter by public_id prefix using a folder filter on folder: In dynamic mode this is non-standard; the system recommends asset_folder.
    Option C — Skip folder filter entirely, filter only by resource_type = raw: Simplest and safest. The flow fires for every raw upload. Acceptable if raw uploads are rare in your environment.
    Option D — Filter by resource_type = raw + a tag prefilter: Have Django add a tag like well-served at upload time; the flow filters on that tag. Most reliable.
