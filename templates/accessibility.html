Feature 3

Accessibility Mode

Bigger targets, frozen layout, no animations — for eye-gaze and switch-scan users.

  • Every button and input grows to a minimum of 80 × 80 px — large enough for eye-gaze activation.
  • The page layout freezes completely — no pop-ups, no sliding banners, no reflows.
  • All animations are switched off — smooth colour changes become instant.
🔲 👁️

Everything on the page gets bigger.

The page stays still. Nothing pops up. Nothing moves.

It is easier to click or look at the buttons.

Turn it on using the "Accessibility Mode" button in the top-right corner of every page.

Blind & screen-reader access

KwaCart is built to work fully with a keyboard and screen reader — no mouse required, no visual-only cues, no unlabelled controls. The list below describes exactly what is in place across every page.

Navigation

Skip-to-content link

Every page has an invisible "Skip to main content" link as its very first focusable element.

  • Press Tab once on any page — the link appears and moves keyboard focus straight past the navigation bar to the main content area.
  • The main content region is marked with a <main> landmark so screen readers can jump to it directly using landmark navigation.
  • The navigation bar is labelled "Main navigation" so screen readers distinguish it from other navigation landmarks on the page.
Forms

Labelled forms & live error announcements

Every form field has an explicit label, and every error is announced aloud the moment it appears.

  • All inputs on the sign-up and log-in pages have visible <label> elements linked by for/id.
  • Validation errors are placed in an aria-live="assertive" region so a screen reader announces them immediately — no hunting for red text.
  • Password requirements and help text are linked to their input via aria-describedby, so they are read out when the field receives focus.
  • Fields with a validation error are also marked aria-invalid="true".
Controls

Contextual control names

Every link and button has a unique, descriptive name — not just "View", "Delete", or "Open" repeated ten times in a row.

  • Archive table actions include the tool name and date — for example "Delete Min Specs submission from 2024-11-14".
  • Both archive tables have a screen-reader-only caption and column headers with scope="col", so table-navigation mode works correctly.
  • Drawing-canvas toolbar buttons (pen sizes S / M / L and the eraser) are labelled "Pen size: small", "Pen size: medium", etc.
  • The tool catalog announces the current mode ("Work Solo" or "Facilitate") as a screen-reader-only page heading when the view switches.
Dynamic content

Live region announcements

Content that changes without a page reload is announced automatically — no visual watching required.

  • The timer announces milestone intervals ("Two minutes remaining", "One minute remaining") via an aria-live region so a screen-reader user always knows how much time is left without looking at the clock.
  • The autosave indicator on solo tool pages announces "Draft saved" each time a save completes.
  • Drawing-canvas actions (pen size change, eraser selected, canvas cleared) are announced via a dedicated aria-live="polite" status region.
